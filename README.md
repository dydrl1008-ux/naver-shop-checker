# 네이버 통검 가격비교 블록 체커 (v2)

키워드를 대량으로 넣으면 각 키워드가 **모바일 통합검색에 "네이버 가격비교" 블록이 실제로 뜨는지** +
**내 nvMid 상품이 그 블록 안 N등(기본 4등) 이내인지**를 확인하는 Vercel용 Next.js 앱.

## 판별 로직 (v1에서 바뀐 점)
- 쇼핑검색 openapi 가 아니라 `m.search.naver.com` **통검 페이지를 직접 스크레이핑**.
- **블록 판별 = `data-slog-container="shp_tli"` 컨테이너 존재 여부.**
  - ⚠️ 텍스트 "네이버 가격비교"는 블록이 **없는** 페이지에도 JSON(`"source":"네이버 가격비교"`)으로
    박혀 있어서 **오탐**난다. 그래서 텍스트 매칭은 폐기하고 shp_tli 컨테이너로만 판별.
- **내 상품 순위** = 블록 카드(`data-slog-content="shp_tli:..."`)를 문서 순서대로 파싱,
  `view_type_guide_{nvMid}` 로 nvMid 매칭. 광고 카드(`nad-` 슬롯)는 광고 제외 순위에서 제외.
- **차단 감지** = status≠200 또는 HTML < 50KB → 캡차/차단 의심 (블록 X 아님).

## 입력 형식
```
복숭아, 7kg 8kg, 핸드크림      ← 콤마/줄바꿈으로 여러 키워드
캠핑의자	82640123456          ← 탭으로 구분하면 그 줄만 해당 nvMid 추적
```
- 공통 nvMid 칸: 줄에 mid 없을 때 적용
- 노출 인정 등수(기본 4) / 광고 제외 토글 / 쿠키(선택) 지원
- 결과 CSV 다운로드

## 실행
```bash
npm install
npm run dev      # http://localhost:3000
```

## Vercel 배포
GitHub push → Vercel Import → Deploy. 환경변수는 필요 없음(쿠키는 선택, UI에서 입력 가능).
차단 잦으면 Vercel 환경변수 `NAVER_COOKIE` 에 세션 쿠키를 넣어두면 기본값으로 쓰임.

## 한계
- 데이터센터 IP(Vercel)는 네이버가 차단/캡차 걸 수 있음 → 쿠키 또는 프록시 경유 필요.
- 통검 블록은 같은 키워드라도 시점/세션/개인화에 따라 떴다 안 떴다 함(복숭아도 케이스마다 다름).
